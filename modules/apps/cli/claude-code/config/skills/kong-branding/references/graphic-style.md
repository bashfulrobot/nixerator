# Kong graphic style (2026 v1.1)

Source: Kong Brand Guidelines PDF, "Design Principles" and "Graphic Style" sections. These are looser than the palette/type/logo rules — treat them as a visual vocabulary to draw from, not a checklist to satisfy exactly.

## Design principles

Three words the guidelines use to frame every visual and product decision:

- **Speed** — Kong frames modern innovation as a continuous, fluid motion rather than the old step-by-step design → build → test → release cycle. Visually: prefer motion, directional lines, and forward-leaning composition over static, boxed-in layouts.
- **Electric** — connectivity that's fast, fluid, and reliable enough to feel invisible — "an extension of the human will," not a visible tool. This is the conceptual backing for why electric lime is the accent color: it's meant to read as a spark, not a background.
- **Architect** — structure, intent, system thinking; designing before building, resolving complexity through clarity. This is the backing for the blueprint/grid graphic language below.

## Blueprint guides

A visual language of lines, grids, and cross-intersections drawn from architectural drafting — construction, precision, intentional design. Concretely:

- **Cross** / **Tee** / **Grid** — line-intersection motifs used as background texture or dividers, not as literal icons.
- **Glow** — a soft lime glow/bloom effect, used sparingly to draw the eye to the one accented element in a composition (echoes the "single accent" color rule).

Use these when you need decorative structure behind content (a hero background, a section divider) rather than inventing an unrelated pattern.

## Connectivity layers

Diagrams showing individual elements (apps, agents, APIs, data stores) operating together as one system — sequential forms arranged in fluid rhythm, expressing harmony through structure. Kong ships both a **dark** and a **light** diagram variant of the same connectivity-layer concept (workflow → agent/AI app → API/event/MCP layer → data), so match the variant to the surface's theme rather than force one styling onto both.

If you're building an architecture diagram for Kong material, this is the reference concept: layered, left-to-right or top-to-bottom flow, each layer clearly separated, connectivity lines rendered in the accent or a light neutral against the dark canvas — not a generic boxes-and-arrows diagram.

## When this matters vs. when it doesn't

For most day-to-day Kong-branded output (a doc, a deck slide, an email signature), the palette + typography + logo rules in the other reference files carry all the weight — graphic style is optional polish. Reach for this file specifically when building a hero graphic, a background texture, or an architecture/connectivity diagram and you want it to feel native to the brand rather than generic.
