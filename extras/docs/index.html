Every feature is a module

Same shape everywhere: an enable option and a config = lib.mkIf cfg.enable block. Drop a default.nix in the right directory — nothing to register.

{ lib, pkgs, config, globals, ... }:

let
  cfg = config.apps.cli.APPNAME;
in
{
  options.apps.cli.APPNAME.enable = lib.mkEnableOption "APPNAME";

  config = lib.mkIf cfg.enable {
    environment.systemPackages = [ pkgs.APPNAME ];
  };
}

Namespace mirrors the directory path: apps/cli/foo/ → apps.cli.foo.enable. apps/gui/one-password/ → apps.gui.one-password.enable (the option name doesn't always match the folder name — read the module, don't assume).

Auto-import, and the archive convention

modules/default.nix recursively imports every default.nix under modules/ via denful/import-tree. No manual import list on workstations.

excluded from auto-import
– disabled/ parked modules
– build/ module-local package derivations
– cfg/ config fragments, local helpers
– reference/ reference material
– archive/ retired, kept for restore
– /_* any path segment starting with _
retiring a module

git mv modules/apps/cli/foo modules/archive/apps/cli/foo

Same relative path under archive/, so restoring later is the same git mv in reverse. 27 modules live there today — decommissioned webapps, superseded desktop tools, and features paused pending an nftables-native rewrite.

How a rebuild happens

just rebuild (or just qr for the quiet/logged variant) runs this, in order:

1
pre-rebuild capture
Live DankMaterialShell settings captured into dank-profiles/ on any host; live Claude Code + Agent OS state captured on qbert only (the designated capture source), so this same rebuild re-seeds them.
2
nixos-rebuild switch --impure --flake .#<host>
The actual build + activation. --impure is needed for the 1Password secrets read at eval time.
3
post-rebuild sync
Skillfish skill updates, a DMS service bounce (skipped if the screen is locked — restarting DMS mid-lock would strand the session), and the same capture flow as step 1 run again to catch anything the rebuild itself changed.
4
auto-commit + push
If capture produced a diff, it's committed and pushed to origin/main automatically — a failed push is non-fatal, just a warning to resolve manually.

Secrets: 1Password, not git-crypt

Nix-eval secrets render from 1Password into a local file before a rebuild reads them — nothing sensitive ever enters the Nix store.

op://nixerator/*→ op inject(secrets.json.tpl)→ ~/.config/nixos-secrets/secrets.json→ read at eval, --impure

This repo used to use git-crypt for secrets in the tree; that was retired in favour of the 1Password + render-secrets flow because it produces secret values (usable in files, env vars, or inline config) rather than just file paths. just render-secrets renders locally; just push-secrets <hosts> pushes the rendered file to headless peers over SSH.

Built to grow

Cross-device Claude Code sessions live

The work fish function + archetypes.claudeWorkHost let a zellij session started on srv (or qbert) be attached from any peer over Tailscale — pick a session up on a different machine, or from a phone. See .claude/docs/cross-device-workflow.md.

Server virtualisation settling

donkeykong, qbert, and srv all moved back to libvirt/KVM after an Incus detour — Talos VMs need real QEMU block-device semantics and a working qemu-guest-agent channel that Incus didn't provide. server.noclaw and server.netbootXyz stay archived rather than nftables-ported, since neither was in active use.
