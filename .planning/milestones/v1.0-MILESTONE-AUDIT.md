## Tech Debt

### Phase 03-hack-workflow

1. **set_phase after worktree removal** (hack.sh line 319): `set_phase("cleanup_done")` writes to `$wt_path/.worktree-state.json` after `git worktree remove --force` has deleted the directory. Suppressed with `2>/dev/null || true`. The final phase is never persisted. Low impact: the worktree is gone, so `handle_existing_worktree` would never trigger for this state.

### Cross-Phase

2. **Duplicated functions**: `check_orphan_worktrees` and `remove_worktree` are defined identically in both github-issue.sh and hack.sh instead of living in lib.sh. Violates NX-03 sharing intent. Any future divergence between copies will be silent.

3. **Dead code**: `create_state` in lib.sh (lines 90-108) is never called. Both scripts define richer variants (`create_issue_state`, `create_hack_state`).

4. **Comment label transposition**: SF-01/SF-02/SF-03 comments in lib.sh are swapped vs milestone requirement definitions. No runtime impact.

5. **Shared SKILL.md**: hack.sh loads `github-issue/SKILL.md` (no hack-specific variant). The SKILL.md contains a "PR Body Format" section irrelevant to hack. Works at runtime but is a semantic mismatch.
