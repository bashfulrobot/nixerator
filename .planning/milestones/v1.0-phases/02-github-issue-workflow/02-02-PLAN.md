**1. `handle_existing_worktree(issue_number, wt_path)`**

When the worktree directory already exists:
- Check state file exists at `${wt_path}/.worktree-state.json`; if not, `die "worktree exists but no state file found at ${wt_path}/.worktree-state.json"`
- Read phase and branch from state file
- Check for merged PR (PM-01): if phase is "pr_created" and pr_url is non-empty, query `gh pr view "$pr_url" --json state --jq '.state' 2>/dev/null || echo "UNKNOWN"`. If result is "MERGED", call `phase_cleanup "$issue_number" "$wt_path"` and return.
- Show compact one-liner: `info "Issue #${issue_number}: phase ${phase}, branch ${branch}"`
- Prompt with `gum choose "Resume" "Remove & restart" "Abort"` (with `|| die "aborted"` for set-e safety)
- Resume: call `phase_resume "$issue_number" "$wt_path" "$phase"`
- Remove & restart: call `remove_worktree "$wt_path"` then `main "$issue_number"` (recursive fresh start)
- Abort: `info "aborted"` and `exit 0`

**2. `phase_resume(issue_number, wt_path, current_phase)`**

Idempotent phase progression. Uses a case statement on current_phase to skip to the next incomplete phase:
- "setup": all setup done, skip to claude_running. Call phase_claude_running, then fall through to remaining phases.
- "claude_running": Claude was interrupted. Try resume with session_id. Call phase_claude_running (it reads session_id from state for --resume), then fall through.
- "claude_exited": call phase_claude_exited, then fall through to push_and_pr.
- "pushing": push may have partially completed. Call phase_push_and_pr (safe_push is idempotent if branch already pushed).
- "pr_created": PR already created. Show `ok "PR already created: $(read_state_field pr_url "$wt_path")"` and exit.
- *): `die "unknown phase: $current_phase"`

The fall-through pattern: after each phase function returns, continue to the next phase function. Use a sequence of if/case guards, not literal bash `;&` fall-through.

Implementation pattern:
```bash
phase_resume() {
  local issue_number="$1" wt_path="$2" current_phase="$3"

  # Register cleanup trap for this worktree
  register_cleanup "$wt_path"

  case "$current_phase" in
    setup|claude_running)
      phase_claude_running "$wt_path"
      ;;&   # fall through to check next
    claude_exited)
      phase_claude_exited "$wt_path"
      ;;&
    pushing)
      phase_push_and_pr "$issue_number" "$wt_path"
      ;;
    pr_created)
      ok "PR already created: $(read_state_field pr_url "$wt_path")"
      return
      ;;
    *)
      die "unknown phase: $current_phase"
      ;;
  esac
}
```

Wait, bash `;;&` continues testing subsequent patterns, which is NOT fall-through. Use explicit sequential calls instead:

```bash
phase_resume() {
  local issue_number="$1" wt_path="$2" current_phase="$3"
  register_cleanup "$wt_path"

  # Determine starting point and run remaining phases
  local start=0
  case "$current_phase" in
    setup)              start=1 ;;
    claude_running)     start=1 ;;
    claude_exited)      start=2 ;;
    pushing)            start=3 ;;
    pr_created)
      ok "PR already created: $(read_state_field pr_url "$wt_path")"
      return ;;
    *) die "unknown phase: $current_phase" ;;
  esac

  if [[ $start -le 1 ]]; then phase_claude_running "$wt_path"; fi
  if [[ $start -le 2 ]]; then phase_claude_exited "$wt_path"; fi
  if [[ $start -le 3 ]]; then phase_push_and_pr "$issue_number" "$wt_path"; fi
}
```

**1. `check_orphan_worktrees()`** (WT-02)

Scan worktree_base for orphaned worktree directories. An orphan is a directory under worktree_base that has NO `.worktree-state.json` file. Called once at startup before any other logic.

```bash
check_orphan_worktrees() {
  local wt_base
  wt_base="$(worktree_base)"
  [[ -d "$wt_base" ]] || return 0

  local found_orphan=0
  local orphan_list=""
  while IFS= read -r -d '' wt_dir; do
    if [[ ! -f "${wt_dir}/.worktree-state.json" ]]; then
      warn "orphan worktree (no state file): $wt_dir"
      orphan_list="${orphan_list}${wt_dir}\n"
      found_orphan=1
    fi
  done < <(find "$wt_base" -maxdepth 1 -mindepth 1 -type d -print0 2>/dev/null)

  if [[ $found_orphan -eq 1 ]]; then
    if gum confirm "Remove orphan worktrees?"; then
      # Remove each orphan individually
      while IFS= read -r -d '' wt_dir; do
        if [[ ! -f "${wt_dir}/.worktree-state.json" ]]; then
          git worktree remove --force "$wt_dir" 2>/dev/null || rm -rf "$wt_dir"
        fi
      done < <(find "$wt_base" -maxdepth 1 -mindepth 1 -type d -print0 2>/dev/null)
      git worktree prune 2>/dev/null || true
      ok "orphan worktrees cleaned"
    fi
  fi
}
```

Note: the `if gum confirm` pattern (SF-04) handles both No (exit 1) and Ctrl+C (exit 130) safely under set -e.

**2. `phase_cleanup(issue_number, wt_path)`** (PM-01 through PM-04)

Post-merge cleanup sequence. Called when handle_existing_worktree detects a merged PR.

```bash
phase_cleanup() {
  local issue_number="$1" wt_path="$2"

  section "Post-merge Cleanup"

  local branch pr_url pr_number default_br
  branch="$(read_state_field branch "$wt_path")"
  pr_url="$(read_state_field pr_url "$wt_path")"
  pr_number="$(printf '%s' "$pr_url" | grep -oE '[0-9]+$')"
  default_br="$(default_branch)"

  # PM-02: switch to default branch and pull
  # Must leave the worktree directory first (can't remove CWD)
  cd "$(git rev-parse --show-toplevel)" || die "cannot cd to repo root"
  git checkout "$default_br"
  git pull origin "$default_br"
  ok "switched to $default_br and pulled"

  # Disable cleanup trap (we're doing intentional removal)
  _WT_CLEANUP_PATH=""

  # PM-03: remove worktree, then prune (WT-05 sequence)
  git worktree remove --force "$wt_path" 2>/dev/null || true
  git worktree prune 2>/dev/null || true
  ok "worktree removed"

  # PM-02: delete local and remote branches (WT-05: after worktree remove + prune)
  git branch -d "$branch" 2>/dev/null || git branch -D "$branch" 2>/dev/null || true
  git push origin --delete "$branch" 2>/dev/null || true
  ok "branches cleaned up"

  # PM-04: comment on issue and PR with resolution summary
  local resolution_msg="Resolved via #${pr_number}. Branch and worktree cleaned up."
  gh issue comment "$issue_number" --body "$resolution_msg" 2>/dev/null || true
  gh pr comment "$pr_url" --body "$resolution_msg" 2>/dev/null || true
  ok "resolution comments posted"

  ok "cleanup complete for issue #${issue_number}"
}
```

Key details:
- The cleanup order follows WT-05: worktree remove, then prune, then branch delete
- `_WT_CLEANUP_PATH=""` disables the trap before intentional removal (Pitfall 5 from research)
- `cd` to repo root before checkout (can't be in worktree dir when removing it)
- Each gh command has `2>/dev/null || true` for idempotent cleanup (already-deleted branches, etc.)
- Resolution comment is the short one-liner format per CONTEXT.md decision
