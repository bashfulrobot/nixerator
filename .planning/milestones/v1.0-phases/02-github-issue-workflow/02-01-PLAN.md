**Helper functions (define at top of script, after argument parsing):**

1. `fetch_issue_metadata(issue_number)` -- calls `gh issue view "$issue_number" --json title,labels,body`, prints JSON
2. `derive_branch_type(labels_json)` -- maps labels to branch types via case statement. Use case-insensitive substring matching on `.name` field (lowercase the label with `${label,,}`). Map: bug->fix, enhancement/feature->feat, documentation/docs->docs, refactor->refactor, test/testing->test, ci->ci, chore->chore, revert->revert, dependencies/deps->deps. Default `*` case uses `gum choose` with `|| die "aborted"` for set-e safety. Process ALL labels with a while loop, return first match; only fall through to gum if NONE match.
3. `build_branch_name(type, number, title)` -- constructs `<type>/<number>-<slug>`. Truncate slug so total branch name stays under ~50 chars. Trim trailing dash from truncation.
4. `create_issue_state(branch, wt_path, issue_number, issue_title, issue_body)` -- extends lib.sh's create_state with issue-specific fields: issue_number, issue_title, issue_body, pr_url (empty string initially). Use jq -n with --arg for each field.

**Phase functions:**

5. `phase_setup(issue_number, wt_path)` -- fetch issue metadata, derive branch type, build branch name, `mkdir -p "$(dirname "$wt_path")"`, `git worktree add "$wt_path" -b "$branch_name"`, `register_cleanup "$wt_path"`, `unlock_git_crypt "$wt_path"`, `create_issue_state(...)`, `set_phase "claude_running" "$wt_path"`. Show ok messages for each step.

6. `phase_claude_running(wt_path)` -- CRITICAL: must `unset CLAUDECODE` in subshell before launching claude. Read issue_body and branch from state file. Build prompt including SKILL.md content (from `$HOME/.claude/skills/github-issue/SKILL.md`), repo context, and issue body. Check for existing session_id for resume. Launch claude in a subshell: `( cd "$wt_path"; unset CLAUDECODE; claude --dangerously-skip-permissions --output-format stream-json -p "$prompt" $resume_flags 2>/dev/null | tee /dev/stderr | jq -r 'select(.type == "system") | .session_id // .sessionId // empty' | head -1 > "/tmp/wf-session-id-$$" ) || true`. After claude exits, read captured session_id from temp file, write to state if non-empty, `set_phase "claude_exited" "$wt_path"`.

7. `phase_claude_exited(wt_path)` -- check if Claude made any changes with `git -C "$wt_path" diff --quiet HEAD` (exit 0 = no changes). If no changes, warn "no changes detected" and exit 0 (not die). If changes exist, `set_phase "pushing" "$wt_path"`.

8. `phase_pushing(wt_path)` -- `( cd "$wt_path" && safe_push "$(read_state_field branch "$wt_path")" )`. After push succeeds, `set_phase "pr_created" "$wt_path"`.

IMPORTANT: The phase name "pr_created" is set BEFORE the PR is actually created. This is deliberate per the research: the phase means "ready to create PR" not "PR exists". The actual PR creation and state update happen together in phase_pr_created. If interrupted between push and PR creation, re-invocation will re-enter phase_pr_created and create the PR.

**Main function:**

main() {
  ISSUE_NUMBER="$1"
  WT_PATH="$(worktree_base)/issue-${ISSUE_NUMBER}"

  # Pre-flight
  assert_clean_tree

  # Fresh start (no existing worktree)
  # Plan 02 will add existing-worktree handling here
  if [[ -d "$WT_PATH" ]]; then
    die "worktree already exists at ${WT_PATH} (resume support coming in next plan)"

  phase_claude_exited "$WT_PATH"

  ok "done! PR created for issue #${ISSUE_NUMBER}"
}
