  <action>
Replace the stub hack.sh with the complete workflow. Follow the github-issue.sh structure closely, adapting for hack-specific behavior. The script has these functions and phases:

**Helper functions:**

1. `create_hack_state(branch, wt_path, description)` -- creates state JSON with `type: "hack"`, `description` field (no issue fields), calls `write_state`. Include fields: type, phase, branch, wt_path, session_id, description, started_at, updated_at.

2. `check_orphan_worktrees()` -- copy from github-issue.sh verbatim (same logic: find dirs without state files in worktree_base, prompt removal via gum confirm).

3. `remove_worktree(wt_path)` -- same as github-issue.sh: clear `_WT_CLEANUP_PATH=""`, read branch from state, force remove worktree, prune, delete branch.

4. `handle_existing_worktree(description, wt_path)` -- adapted from github-issue.sh: check for state file or die, read phase+branch, print info line ("hack: phase X, branch Y"), gum choose Resume/Remove/Abort. Resume calls `phase_resume`, Remove calls `remove_worktree` then `main "$description"`, Abort exits 0.

5. `phase_resume(description, wt_path, current_phase)` -- numeric start index pattern:
   - setup/claude_running -> start=1
   - claude_exited/diff_review -> start=2 (per locked decision: always re-show diff on resume from diff_review)
   - merged/cleanup_done -> print "already merged", return
   - unknown -> die
   - Register cleanup, then run phases from start index. After all phases, set `_WT_CLEANUP_PATH=""`.

**Phase functions:**

6. `phase_setup(description, slug, wt_path)` -- section "Setup", build branch name `hack/${SLUG}`, mkdir -p parent, git worktree add, register_cleanup, unlock_git_crypt, create_hack_state, set `_WT_CLEANUP_PATH=""` (survive interruption after setup), set_phase claude_running.

7. `phase_claude_running(wt_path)` -- section "Launching Claude", read description+branch from state, load SKILL.md from `$HOME/.claude/skills/github-issue/SKILL.md` (same skill per locked decision), build prompt: "You are working in worktree {wt_path} on branch {branch}.\n\nTask: {description}\n\n{skill_content}", check for existing session_id for --resume flag, launch claude in subshell (cd wt_path, unset CLAUDECODE, `claude --dangerously-skip-permissions --output-format stream-json -p "$prompt" $resume_flags 2>/dev/null | tee /dev/stderr | jq -r 'select(.type == "system") | .session_id // .sessionId // empty' | head -1 > "/tmp/wf-session-id-$$"`) with `|| true`, capture session_id and write to state, set_phase claude_exited.

8. `phase_claude_exited(wt_path)` -- section "Checking Changes", check for changes using same pattern as github-issue.sh: `git status --porcelain | grep -v state file`, `git diff --quiet HEAD`. If no changes, warn and exit 0. Otherwise ok "changes detected", set_phase diff_review.

9. `phase_diff_review(wt_path)` -- section "Diff Review", read branch+description from state, get default_br. Show diff: `git -C "$wt_path" diff --color=always "${default_br}...${branch}" | gum pager`. Then gum confirm "Merge to ${default_br}?":
   - If approved: call `phase_merge "$wt_path" "$branch" "$default_br"`
   - If rejected or Ctrl+C: set `_WT_CLEANUP_PATH=""` (preserve worktree), print "merge rejected -- worktree preserved at ${wt_path}", print "resume: hack \"${description}\"" (copy-pasteable per locked decision), exit 0.

10. `phase_merge(wt_path, branch, default_br)` -- section "Merging":
    - Get repo root: `git -C "$wt_path" rev-parse --show-toplevel`
    - cd to repo root
    - `git checkout "$default_br"`
    - Set `_WT_CLEANUP_PATH=""` (disable trap before intentional removal)
    - `if ! git merge --ff-only "$branch"; then` warn ff failed, warn worktree preserved for manual resolution, exit 1
    - ok "merged"
    - WT-05 cleanup: `git worktree remove --force "$wt_path" 2>/dev/null || true`, `git worktree prune 2>/dev/null || true`, ok "worktree removed"
    - `git branch -d "$branch" 2>/dev/null || git branch -D "$branch" 2>/dev/null || true`, ok "branch deleted"
    - `set_phase "cleanup_done" "$wt_path" 2>/dev/null || true` (best-effort, dir may be gone)

**Main function:**

11. `main(description)` -- parse description arg, slugify, compute WT_PATH as `$(worktree_base)/hack-${SLUG}`. Call assert_clean_tree, check_orphan_worktrees. If WT_PATH exists, call handle_existing_worktree and exit 0. Otherwise run: phase_setup, phase_claude_running, phase_claude_exited, phase_diff_review. After diff_review returns (approve path completed inline), set `_WT_CLEANUP_PATH=""`, ok "done!".

**Entry point:**

Keep existing --help/-h and arg validation at top. Call `main "$DESCRIPTION"` at end.

**Anti-patterns to avoid (from RESEARCH.md):**

- Never bare `gum confirm` -- always `if gum confirm ...; then`
- Always `_WT_CLEANUP_PATH=""` before intentional worktree removal AND before exit on reject path
- Run `git merge` from repo root via `git rev-parse --show-toplevel`, NOT from worktree
- Use three-dot diff (`...`) not two-dot (`..`)
- Do NOT add `_WT_CLEANUP_PATH=""` before the merge failure exit 1 (it is already cleared before the merge attempt)
