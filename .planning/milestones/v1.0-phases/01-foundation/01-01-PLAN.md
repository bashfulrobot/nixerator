Create `modules/apps/cli/worktree-flow/scripts/lib.sh` containing all shared functions. This file is inlined at Nix build time via string interpolation, so it must NOT contain `set -euo pipefail` (writeShellApplication injects that). Add a comment at the top: `# NOTE: set -euo pipefail injected by writeShellApplication`.

Include these function groups in order:

**1. Color helpers** (copy from gcmt.sh verbatim):
- RED, GREEN, YELLOW, CYAN, NC color variables
- `info()`, `ok()`, `warn()`, `die()` functions using printf

**2. Section header** (per CONTEXT.md locked decision):
- `section()` function using `gum style --bold --foreground 6 "-- $* --"` with newlines before/after

**3. Safety guards** (SF-01, SF-02, SF-03):
- `assert_not_main()`: reads current branch via `git rev-parse --abbrev-ref HEAD`, dies if main or master
- `assert_clean_tree()`: uses `git status --porcelain`, dies if non-empty with message "working tree is not clean -- commit or stash first"
- `safe_push()`: calls `assert_not_main`, then `git push -u origin "$1"` (takes branch name as arg)

**4. git-crypt auto-unlock** (SF-05, per CONTEXT.md: no interactive picker):
- `unlock_git_crypt()`: takes WT_PATH as `$1`. Checks `$HOME/.ssh/git-crypt` dir exists, finds first key with `find "$key_dir" -maxdepth 1 -type f | head -1`, skips silently if no dir or no keys. On key found: `info "unlocking git-crypt..."`, runs `git -C "$1" crypt unlock "$key"`, verifies with `git -C "$1" crypt status >/dev/null 2>&1 || die "git-crypt unlock verification failed"`, then `ok "git-crypt unlocked"`.

**5. Default branch detection** (from gcom pattern):
- `default_branch()`: uses `git symbolic-ref refs/remotes/origin/HEAD | sed 's|refs/remotes/origin/||'`

**6. Atomic state file I/O** (WT-04):
- `write_state()`: takes JSON string as `$1` and WT_PATH as `$2`. Creates tmpfile with `mktemp "${2}/.worktree-state.XXXXXX"`, writes `$1` to tmpfile, `mv` to `${2}/.worktree-state.json`.
- `create_state()`: takes type ($1: "issue"/"hack"), branch ($2), wt_path ($3). Generates timestamp with `date -u +"%Y-%m-%dT%H:%M:%SZ"`. Builds JSON with `jq -n` using `--arg` for each field: type, phase ("setup"), branch, wt_path, session_id (""), started_at, updated_at. Calls `write_state`.
- `set_phase()`: takes new_phase ($1) and wt_path ($2). Reads current state, updates phase and updated_at via `jq --arg`, calls `write_state`.
- `read_state_field()`: takes field name ($1) and wt_path ($2). Uses `jq -r ".$1"` on the state file.

**7. Trap cleanup handler** (WT-03):
- `cleanup()`: takes no args but uses global `_WT_CLEANUP_PATH`. If set and directory exists: `warn "cleaning up worktree at $_WT_CLEANUP_PATH..."`, runs `git worktree remove --force "$_WT_CLEANUP_PATH" 2>/dev/null || true`, then `git worktree prune 2>/dev/null || true`.
- `register_cleanup()`: takes wt_path as `$1`. Sets `_WT_CLEANUP_PATH="$1"`, then `trap cleanup EXIT INT TERM`.
- Initialize `_WT_CLEANUP_PATH=""` at module level.

**8. Slug generation** (using bash built-ins per RESEARCH.md recommendation, avoiding gnused dep):
- `slugify()`: takes string as `$1`. Lowercase with `${1,,}`, replace non-alphanumeric with dash using `tr -cs 'a-z0-9' '-'`, trim leading/trailing dashes with `sed 's/^-*//;s/-*$//'`, truncate to 50 chars.
  NOTE: This uses `tr` (from coreutils) and `sed`. If gnused causes issues, this is pure POSIX-compatible sed usage (no GNU extensions). Per STATE.md blocker resolution: keeping gnused in runtimeInputs for now but the sed usage here is POSIX-safe.

**9. Worktree path helper**:
- `worktree_base()`: outputs `"$(git rev-parse --show-toplevel)/../.worktrees"`.

All gum confirm usage MUST use the `if gum confirm; then ... else ... fi` pattern (SF-04). In this file there are no direct gum confirm calls (those are in the command scripts), but document the pattern in a comment for reference.
